1. Platform overview

Shifa is a complete direct-to-consumer commerce platform for a herbal-remedies business, built as two connected applications sharing one secure backend and database:

  • Storefront — a fast, mobile-first Progressive Web App (installable, works partly offline) where customers browse products, place orders, pay, and track delivery.
  • Admin / OMS — the staff control room: order management, packing, courier dispatch, money reconciliation, catalog, inventory, procurement, finance, reports and settings.

Both run on a single secure server with a shared MySQL database. Everything a customer does on the store appears instantly for staff on the admin panel, and vice-versa.

2. Roles & who does what

Access is role-based — each staff member only sees the sections relevant to their job. This keeps the panel simple and protects sensitive money/customer data.

3. Customer storefront features

Discover & browse

  • Product catalog with categories & collections, search, filters and sorting.
  • Rich product pages with images, description, price/offer, stock awareness and related products.
  • Verified customer ratings & reviews (moderated by admin).
  • SEO-optimised: clean URLs, meta/OpenGraph tags, product structured data, sitemap — built to rank on Google.

Shop & buy

  • Wishlist with a shareable public link, and a persistent cart that follows the customer across devices when logged in.
  • Checkout with pincode → city/state autofill, live inline validation, and a delivery-date estimate.
  • Payment options: Cash on Delivery (COD) and online payment (gateway-ready).
  • Coupons & offers with live discount preview in the cart.
  • Instant order confirmation by email and a WhatsApp handoff for support.

Track & engage

  • Order tracking by order code, with a live order-status chat/timeline.
  • Customer accounts with full order history & re-order.
  • Branded transactional emails: welcome, order confirmed, shipped, delivered.
  • Experience polish: cart drawer, add-to-cart animations, toasts, sticky mobile buy button, dark mode, English/Hindi, installable app with an offline page.

4. Admin / OMS features

Command centre

  • Dashboard with live KPIs and charts (revenue trend, orders by status, top products/states) and a real-time activity feed that updates without refreshing.
  • Global search and saved filter views.

Order operations

  • Orders list with server-side pagination, sorting, filtering and bulk actions (approve, mark-packed, merged label PDF).
  • Approval queue for reviewing and approving new orders.
  • New Order entry for salespeople/admin, with a per-line price override and a screenshot upload.
  • Packing scan station that generates shipping + internal labels and barcodes.
  • Returns / RTO workflow: initiate return, capture reason, restock, track refund.

Money & catalog

  • Reconciliation of COD collections and courier loss/damage claims.
  • Branded invoices (plain & GST tax invoice) and reports (Excel / PDF / Vyapar export) in the Shifa theme.
  • Products (with CSV bulk import), inventory with a stock-movement ledger & low-stock alerts, coupons and review moderation.

Procurement, finance & governance

  • Procurement: suppliers & purchase orders; receiving goods automatically restocks inventory.
  • Finance: expenses tracking and a Profit & Loss view (revenue vs costs vs expenses).
  • Customers / CRM: lifetime value, order history, repeat-buyer flags.
  • Notifications centre + bell, global audit log (who changed what, when).
  • Settings: company identity & logo, GST config, invoice numbering, bank details, terms, tax slabs.
  • User management with role assignment.
  • Scheduled daily sales-digest email to the owner.

5. Customer journey (store flow)

Land on store → Browse / search / filter → Open product → Add to wishlist / cart → (optional) apply coupon → Checkout: enter details (pincode autofill) → Choose COD or Online payment → Place order → Instant confirmation (email + on-screen + WhatsApp) → Track order status → Delivered → Leave a review

A logged-in customer keeps their cart, wishlist and full order history across devices. A guest can still check out and track by order code.

6. Order lifecycle (admin flow)

Every order moves through a controlled state machine. Staff actions and courier updates advance the status; nothing skips a step, so the records always stay trustworthy.

7. COD & reconciliation flow

Delivered (COD) → Courier collects cash → Courier remits to business → Accountant records "received from courier" → Receivable settled Lost/Damaged → Loss claim raised → Claim tracked → Recovered / written-off

The reconciliation ledger always shows what money is still owed by the courier (COD not yet remitted) and the status of any loss claims — this feeds the Profit & Loss view.

8. Returns / RTO flow

Return requested / RTO → Reason captured → Goods received back → Inventory restocked → Refund tracked (if prepaid) → Return closed

9. Procurement & stock flow

Create Supplier → Raise Purchase Order (items, qty, unit cost) → Goods arrive → Receive PO (full or partial) → Inventory auto-restocked → Stock ledger updated → Expense recorded

Selling decrements stock automatically; receiving a PO increments it. Low-stock alerts warn before items run out.

11. Technology & security

  • Frontend: Angular (two apps: storefront + admin), installable PWA, responsive, light/dark, EN/HI.
  • Backend: Java / Spring Boot REST API with a MySQL database and versioned schema migrations.
  • Security: token-based (JWT) login, role-based access control, BCrypt-hashed passwords, per-role method-level authorization, audit logging.
  • Integrations (swappable): payment gateway, courier API + webhooks, WhatsApp Business API, SMTP email — each runs in a safe "mock/sandbox" mode until real credentials are switched on.
  • Reliability: nightly database backups, transactional outbox for guaranteed notification delivery, real-time dashboard via server-sent events.
  • Hosting: single cloud VM with Nginx + the app + MySQL; HTTPS-ready; low running cost.
